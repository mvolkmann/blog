Overview

TestFlight enables deploying test versions of an iOS app that does not require a full app review. New versions can be deployed without a followup review. Each version remains available for download for 90 days.

After all issues identified by testers have been resolved, you are ready to submit the app for a full review before it is added to the App Store. TODO: Add documentation on the steps to submit to the App Store.

It is possible to get an app featured by Apple on the App Store. See Getting Featured on the App Store. In order to be featured you must tell Apple about your app by completing the form at Tell us about your app or game.

Steps

Part 1 - Creating an Identifier

  1. Browse developer.apple.com and sign in.
  2. In the top nav bar, click "Account".
  3. In the "Certificates, Identifiers & Profiles" section, click "Identifiers".
  4. To create a new identifier, click the "+" after "Identifiers".
  5. Select "App IDs".
  6. Click the "Continue" button.
  7. Click the "App" button.
  8. Click the "Continue" button.
  9. Enter an app description.
  10. Open the app in Xcode.
  11. Select the top entry in the Navigator.
  12. Select the first entry under TARGETS.
  13. Click the General tab.
  14. Copy the app bundle identifier from Xcode and paste it in the "Bundle ID" field.
  15. Check the desired "Capabilities" such as "HealthKit".
  16. Click the "Continue" button.
  17. Click the "Register" button.

Part 2 - Creating an App

  1. Browse appstoreconnect.apple.com.
  2. Click the large "My Apps" button.
  3. Click the "+" button after "Apps".
  4. Select "New App".
  5. Check the platforms the app supports such as "iOS".
  6. Enter a name for the app that is unique across all apps in the app store.
  7. Select a language such as "English (U.S.)".
  8. Select the bundle ID for the app that was created in part 1.
  9. Enter a SKU with no spaces that is unique among your apps.
  10. Choose between limited (restricted set of test users) and full access (any user can access).
  11. Click the "Create" button.
  12. If a spinner is displayed for a long time, refresh the browser.
  13. The "App Store" tab is selected by default and all the fields on this page must be completed to deploy to the app store. But to just deploy to TestFlight, click the "TestFlight" tab.

Part 3 - Creating a Build

  1. Open the app in Xcode.
  2. Select the top entry in the Project Navigator.
  3. Select the first entry under TARGETS.
  4. Click the General tab.
  5. Set "Version" to "0.1.0" for the initial build which is a semantic version number. Typically this is modified each time new features or bug fixes are being deployed but changing this is not required. An advantage of only incrementing the build number is that a new review of the app will not be triggered. (It seems I had to set all the "MARKETING_VERSION" values to "1" it order to successfully upload a build.)
  6. Set "Build" to "1". Increment this for each subsequent build. It does not need to match the value of "Version".
  7. After "App Icon", enter "AppIcon" which should be the asset name of the application icon.
  8. After "App Icon Source", check the "Include all app icon assets" checkbox.
  9. Repeat the previous two steps for each target using matching values.
  10. Verify that the file "Assets.xcassets" defines the image set "AppIcon" that contains icons for both iPhone and iPad (if building for both).
  11. If a paired watchOS app is included, the watch app will have its own "Assets.xcassets" file that must contain "AppIcon" for the watch.
  12. In the device list at the top, choose "Any iOS Device".
  13. Select Product ... Archive. This can take a few minutes to complete on a fast machine.
  14. In the dialog that appears, click the "Distribute App" button. This dialog may be on a different screen and behind other windows.
  15. Select the "App Store Connect" radio button.
  16. Click the "Next" button.
  17. Select the "Upload" radio button.
  18. Click the "Next" button.
  19. In the dialog that appears, uncheck "Manage Version and Build Number". (If you get the error "Metadata/Info.plist Mismatch" when uploading the archive, try again with this checkbox checked.)
  20. Click the "Next" button.
  21. Select the "Automatically manage signing" radio button.
  22. Click the "Next" button.
  23. If an "Apple Distribution certificate" has not yet been created:
    • Check "Generate an Apple Distribution certificate".
    • Press the "Next" button.
    • Press the "Export Signing Certificate..." button and save it somewhere on your hard drive.
    • Click the "Next" button.
  24. Click the "Upload" button. This can take a few minutes to complete on a fast machine.
  25. You will be asked if the app uses encryption, such as sending HTTP requests using HTTPS. For apps that do not use encryption, the question can be suppressed by adding the Boolean key "App Uses Non-Exempt Encryption" in the target Info tab with a value of "NO". For apps that do use encryption, you will have to submit a form to the U.S. government which is explained in the encryption questions. After answering the questions displayed in a dialog, click the "Start Internal Testing" button. The status should change to "Ready to Submit"

Encryption

By default every time an app is submitted there will be a prompt related to use of encryption. A dialog will appear with the title "Export Compliance Information" and the question "Does your app use encryption?". To avoid the need to answer this question every time a new version of the app is released:

  • Select the top entry in the Navigator.
  • Select the main target.
  • Click the "Info" tab.
  • Click the "+" button in any row.
  • Add the key "App Uses Non-Exempt Encryption" with the value "YES" or "NO".

Part 4 - Distributing an App

  1. Return to the App Store Connect web page.

  2. Login with your Apple ID.

  3. Click the large "My Apps" button.

  4. Click the app to be distributed.

  5. Click the "TestFlight" tab.

  6. If the build to be distributed has a STATUS of "Processing", refresh the page periodically until that changes.

  7. If the build to be distributed has a STATUS of "Missing Compliance", click the "Manage" link and answer the questions asked. The status should change to "Ready to Submit".

  8. Click the build number to be distributed (ex. 4).

  9. If this is the first time distributing this app ...

    1. In the "App Previews and Screenshots" section, add at least three screenshots for each supported device size.

      These must have specific pixel dimensions. A simulator that provides screenshots for a 6.5" iPhone is "iPhone 11 Pro Max". A simulator that provides screenshots for a 5.5" iPhone is "iPhone 8 Plus". A simulator that provides screenshots for a 12.9" iPad is "iPad Pro 6th generation".

      To capture screenshots, run the app in the Simulator once for a device that has the desired size. Click the camera button in the upper-right or press cmd-s. This displays a screenshot thumbnail in the lower-right. Right-click the thumbnail and select "Save to Desktop".

      On the App Store Connect page, click "Prepare for Submission" in the left nav. Drag the screenshot files into the area under "Version Information".

    2. Click the "Create Group" link in the light blue rectangle to add testers.

    3. Enter a group name.

    4. Check the "Enable automatic distribution" checkbox.

    5. Click the "Create" button.

    6. Click "Test Information" under "General Information" in the left nav.

    7. Complete the ENTIRE form.

    8. If the app does not require sign in, check and uncheck that checkbox. Otherwise the form will not be considered complete.

    9. Click the "Save" button in the upper-right.

  10. Click the circled "+" after "Group".

  11. In the dialog that appears, check at least one group name.

  12. Click the "Next" button.

  13. Enter a description of what testers should test.

  14. Click the "Submit for Review" button.

  15. Click the back link near the top of the page that says "iOS Builds".

  16. Verify that the status changed to "Waiting for Review". Testers will receive an email about the new version in a day or two. The status of this build will change to "Testing".

Part 5 - App Configuration

  1. Return to the App Store Connect web page and the "App Store" tab.
  2. If the page for the app is not already displayed, click the large button containing the app name.
  3. Visit each section linked in the left nav and supply any missing information. There are many things that must be specified, especially under "App Privacy"! For each section, click the "Save" button after entering all the required data.
  4. Click "Prepare for Submission" in the left nav.
  5. Click the "Add for Review" button in the upper-right.
  6. If you see "Unable to Add for Review", supply all the missing information that is listed.

Part 6 - Adding External Testing Users

  1. Click the "+" after "External Testing" in the left nav.

  2. Enter a group name.

  3. Click the "Create" button.

  4. Under "External Testing" in the left nav, click the new group name.

  5. For each external testing user:

    • Click the "+" after "Tester".
    • Select "Add New Testers".
    • Enter their email, first name, and last name.
    • Click the "Add" button.

  6. Click the "+" after "Build" and select a build to be tested.

  7. Does something else need to be clicked to save all of this?

  8. The status of the selected build should change to "Waiting for Review".

  9. Wait for an approval email from Apple, after which the testers should receive an email containing a link that will allow them to download the app from TestFlight.

  10. If a build of the app is rejected, it will display a status of "Rejected" on the "App Store Connect" web page. To see the reasons for the rejection:

    • Click the "App Store" link in the top nav.
    • Click "App Review" in the left nav.
    • Click the "Resolve" link on the far right.

Test users must install the TestFlight app on their devices in order to download test builds of the app and provide feedback.

Part 7 - Submit for Review

  1. Return to the App Store Connect web page and the "TestFlight" tab.
  2. In the table, click the build number link for the version that was just uploaded.
  3. Address issues, if any, identified in the "STATUS" column.
  4. Wait for the status to change to "Ready to Submit".
  5. Click the "+" to the right of "Groups".
  6. If there are no tester groups, add at least one.
  7. Select at least one group to test the new app build.
  8. Click the "Save" button in the upper-right? VERIFY THIS AGAIN!
  9. Click the "Submit for Review" button? VERIFY THIS AGAIN!
  10. Return to the previous page that lists versions of the app and note that the submitted version now has a status of "Waiting for Review".
  11. It can take a couple of days for Apple to review the new build.

Part 8 - Resolve App Review Issues

  1. Return to the App Store Connect web page and the "App Store" tab.
  2. Click the large button for the app. It seems that having the app in TestFlight is not enough to cause its icon to be displayed here. The app may need to be submitted to the App Store in order to display its icon on this web page.
  3. In the left nav, click "App Review".
  4. If the "REVIEW STATUS" column shows "Rejected", click the "Resolve" link at the end of the row.
  5. Read the messages explaining why the app was rejected.
  6. Address all of the issues and verify that they are no longer present when running the app locally.
  7. In the left nav, select the first entry under "iOS App".
  8. In the main area, scroll down to the "Build" section.
  9. Click the "-" button to disassociate the previous build.
  10. Click the "+" button after the "Build" section heading.
  11. Select a new build.
  12. Click the "Save" button in the upper-right.
  13. Click the "Add for Review" button in the upper-right.
  14. A "Confirm Submission" page will be displayed.
  15. Click the "Submit to App Review" button in the upper-right.
  16. The app status will change to "Waiting for Review".
  17. Wait for feedback from Apple. This could take a couple of days.

Part 9 - Instructions for Test Users

  1. You receive an email with the subject "You've been invited to the App Store Connect".
  2. View this email on your mobile device4.
  3. Click the "Accept Invitation" link in the email.
  4. A new page opens in mobile Safari.
  5. Enter your Apple ID and password to login.

Part 10 - Sales

To see statistics on app sales, browse App Store Connect and click "Sales and Trends".

Feedback
